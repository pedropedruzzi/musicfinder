\chapter{Melhorias e Trabalhos futuros}

Desde o início a proposta do trabalho foi desenvolver uma técnica de comparação de conteúdos musicais utilizando técnicas adaptativas, associada a um protótipo funcional que fosse capaz de exercitar e avaliar a técnica de comparação desenvolvida. Assim, diversos pontos do trabalho foram abordados de forma superficial e são passíveis de grandes melhorias. A seguir serão apresentados alguns dos pontos de extensão mais importantes.

\paragraph*{Automatização da identificação de notas.} Parte deste processo foi executada de forma manual e é passível de automatização. Para tanto a análise das variações de intensidade e frequência ao longo da linha de tempo podem ser extremamente úteis. Pode-se utilizar como base trabalhos já desenvolvidos, a fim de melhorar este componente do protótipo.

\paragraph*{Maior granularidade de erros.} Apenas três tipos de erros foram considerados para este trabalho (\emph{OMISSÃO}, \emph{ADIÇÃO} e \emph{TROCA} de uma nota). Porém, ao longo do desenvolvimento percebeu-se que com apenas estas classes de erro não é possível avaliar com precisão a intensidade do desvio entre as melodias comparadas. Isto ocorre pelo fato de que o caso de erro \emph{TROCA} permite a substituição por qualquer outra nota. Assim sugere-se a criação de uma nova classe de erro, denominada \emph{VARIAÇÃO MENOR}, similar a uma \emph{TROCA} porém representando um desvio menor de nota, dentro de uma tolerância pré-determinada. Esta nova classe tem o objetivo de descrever um erro muito comum, em que há um pequeno desvio com relação à nota original.

\paragraph*{Múltiplos erros.} O autômato desenvolvido é capaz de lidar com erros simples apenas (dentro da definição proposta). Mais de um erro pode ser aceito desde que haja um espaçamento mínimo entre estes. Porém é relativamente comum, em especial para o caso de \emph{TROCA}, que ocorram dois erros seguidos. O projeto atual do autômato não é capaz de lidar com esta situação, gerando \emph{scores} de proximidade muito mais baixos do que o esperado para trocas seguidas.

\paragraph*{Definição de pesos de cada classe de erro.} Para o cálculo da similaridade, diversos fatores devem considerados, como:

\begin{itemize}
\item Número de erros
\item Tipos de erros cometidos
\item Aceitação ou não da cadeia (atingir o fim do autômato)
\end{itemize}

Tais fatores influenciam diferentemente na similaridade entre dois trechos. A omissão de uma nota, por exemplo, é intuitivamente um erro mais grave do que uma pequena troca de nota (\emph{VARIAÇÃO MENOR}). Porém, não é feita distinção entre estes dois casos. Testes com repositórios maiores e associados a um \emph{feedaback} humano poderia prover dados que ajudassem a calibração dos pesos de cada um destes fatores no \emph{score} final.

\paragraph*{Generalização do método de comparação.} O método comparação descrito independe do fato de a cadeia de entrada ser proveniente especificamente de uma sequência de notas. Assim, este método pode ser generalizado para lidar com cadeias de símbolos de qualquer natureza. Seria necessário generalizar todos os conceitos envolvidos, como alfabeto de entrada, classes de erro aceitáveis, etc. Este método de comparação seria apropriado para busca padrões em cadeias, acrescentando uma flexibilização maior do que uma busca exata ou uma expressão regular.
